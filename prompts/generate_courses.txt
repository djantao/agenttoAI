一、课程规划表输出规范（必须包含以下6列内容）：
1. 序号：阿拉伯数字，从1开始递增
2. 课程名称：简洁无冗余，突出核心内容（如：会计科目实操 / Python 基础语法）
3. 课程简介：≤80字，说明课程在模块中的核心作用，贴合学习目标
4. 适合难度：仅限初级/中级/高级三选一，精准匹配用户基础
5. 所属模块：名称明确无歧义（如：会计入门核心 / Python 基础进阶 / UI 设计原理）
6. 对应章节（含核心目标）：每门课程拆分为2-3个章节，每个章节标注核心目标，逻辑遵循「认知→方法→应用」的通用学习规律

二、通用课程设计要求：
- 内容适配性：课程内容需贴合对应领域的最新行业标准、技术趋势或知识体系，避免过时内容
- 理论实操平衡：根据领域特性调整理论与实操占比（如技术类侧重实操、文科类侧重理论+案例），实操类课程需明确具体任务目标
- 难度递进逻辑：课程按「基础→进阶→实战」顺序排列，模块划分清晰，同一模块内课程难度连贯无断层
- 适配Notion：表格可直接复制粘贴到Notion，无复杂格式，支持一键转为Notion数据库
- 领域灵活性：可适配任意学习领域，从职业技能（财务、编程）到兴趣爱好（书法、摄影）均能精准规划

三、输出格式要求：
请同时提供两种输出格式：
1. 首先输出纯文本表格格式，便于直接复制到Notion
2. 然后输出标准JSON格式，JSON字段与表格列名一一对应，数组内每个元素为单门课程的完整信息

JSON格式要求：
- 根对象包含一个courses数组
- 每个course对象包含以下字段：
  - serialNumber（序号）
  - courseName（课程名称）
  - courseDescription（课程简介）
  - difficulty（适合难度）
  - module（所属模块）
  - chapters（对应章节，数组类型，每个章节包含title和coreObjective字段）
